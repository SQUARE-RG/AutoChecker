# 软件部署流程

## 目录树
使用如下命令将软件克隆到本地：
```shell
git clone https://github.com/Carlson-JLQ/code_check.git
git checkout web-release
cd code_check
```
进入项目根目录后，可以看到如下目录树：

+ AutoChecker ---------------------------------软件根目录
  + llvm-project ----------------------------llvm本地仓库
  + clang_tidy_collect --------收集meta-OP数据库和API数据库
    + check_op.json -------------check函数相关meta-OP数据库
    + astMatcher_op.json ------astMatcher相关meta-OP数据库
    + all_astMatchers.json --------astMatcher相关API数据库
    + all_check_api.json ----------check函数相关API数据库
  + test_case ------------------------------测试用例目录
    + test_case_1.cpp
    + ......
  + result-generation --------------------软件输出目录
    + first_checker ---------------第一阶段生成的检查器代码
      + checker.cpp
      + checker.h
    + final_checker ------------------最终生成的检查器代码
      + checker_final.cpp
      + checker_final.h
  + src
    + ......
    + main.py ----------------------软件入口
    + config.json ------------------软件配置文件
  + rule.json ---------------------------------软件输入
  + requirements.txt


软件运行需要依赖python环境并提前编译clang-tidy等工具，做好准备工作后，需要按照要求修改输入文件rule.json并提供相关测试用例套件。
## 环境准备
准备软件运行需要的python环境，并且提前编译clang-tidy，软件未来会支持生成Codeql的query文件，所以也需要提前准备Codeql的环境。
### python 环境准备
```shell
sudo bash scripts/prepair_python_env.sh
```
下载安装miniconda,创建虚拟环境autochecker，安装requirements.txt中的依赖包。

### 编译clang-tidy
需要提前编译Clang-Tidy，请依次执行如下命令完成编译过程：
```shell
sudo bash scripts/prepair_clang_tidy.sh
```
### 准备Clang-tidy开发流
```shell
bash scripts/clang_tidy_dev_flow.sh
```


```shell
cd /root/code_check/llvm-project/clang-tools-extra/clang-tidy


mkdir ucassaat
mkdir ../test/clang-tidy/checkers/ucassaat

```

编辑 /root/code_check/llvm-project/clang-tools-extra/clang-tidy 目录下的 CMakeLists.txt 文件，找到以下语句并在上方插入对应语句。
```txt
add_subdirectory(ucassaat)
# 找到下一行语句，并在其上一行插入第一行语句
add_subdirectory(zircon)
```

```txt
  clangTidyUcasSaatModule
# 找到下一行语句，并在其上一行插入第一行语句
  clangTidyZirconModule
```


编辑 /root/code_check/llvm-project/clang-tools-extra/clang-tidy 目录下的 ClangTidyForceLinker.h 文件，找到以下语句并在上方插入对应语句。
```cpp
// This anchor is used to force the linker to link the UcasSaatModule.
extern volatile int UcasSaatModuleAnchorSource;
static int LLVM_ATTRIBUTE_UNUSED UcasSaatModuleAnchorDestination =
    UcasSaatModuleAnchorSource;

/** 找到以下（约 135~138 行附近），并在其上一行插入上方的语句 **/
// This anchor is used to force the linker to link the ZirconModule.
extern volatile int ZirconModuleAnchorSource;
static int LLVM_ATTRIBUTE_UNUSED ZirconModuleAnchorDestination =
    ZirconModuleAnchorSource;
```

切换到/root/code_check/llvm-project/clang-tools-extra/clang-tidy/ucassaat目录，将/root/code_check/scripts/clang-tidy-work-flow/CMakeLists.txt复制到该目录下
将/root/code_check/scripts/clang-tidy-work-flow/UcasSaatTidyModule.cpp复制到该目录下
将/root/code_check/scripts/clang-tidy-work-flow/HelloWorldCheck.h复制到该目录下
将/root/code_check/scripts/clang-tidy-work-flow/HelloWorldCheck.cpp复制到该目录下


回到 LLVM 源码根目录/root/code_check/llvm-project，输入以下命令编译：
```shell
/usr/bin/cmake --build /root/code_check/llvm-project/build --target clang-tidy -j
```
接着执行如下命令：
```shell
/root/code_check/llvm-project/build/clang-tidy --list-checks --checks=-*,ucassaat-*
```
截取这个命令的输出，如果有输出，说明开发流完成。
